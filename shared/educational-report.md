# Educational Change Report

Every engineering run that changes an artifact should teach the user what improved.

Use this compact report:

```markdown
## Original Strengths
- ...

## Original Weaknesses
- ...

## Changes Made
- ...

## Why This Improves Local-Model Reliability
- ...

## Remaining Risks or Assumptions
- ...
```

Keep it concise. Focus on practical lessons, not praise or filler.
