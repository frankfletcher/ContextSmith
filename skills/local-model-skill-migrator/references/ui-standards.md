# UI Standards for Agent Instructions

Use this reference when the repo contains frontend/UI code or the user asks for UI work.

## Baseline UI Standards

- Follow the existing design system, components, spacing, typography, and copy style.
- Preserve accessibility: labels, semantic HTML, keyboard navigation, focus states, contrast, and screen-reader compatibility.
- Prefer consistent layout over novelty.
- Avoid adding animations unless they serve usability.
- Keep copy concise and professional.
- Do not use emojis in production UI, code, logs, documentation, or commit messages unless the project already uses them or the user explicitly requests them.
- Validate responsive behavior when relevant.
- Do not introduce new UI libraries without approval.

## Repo Scan

Before adding UI standards, inspect:

- component library/design system usage
- styling framework/config
- accessibility/test setup
- existing copy tone
- screenshots or design docs when available
